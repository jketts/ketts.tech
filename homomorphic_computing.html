A Quantum Approach to Homorphic Computing

So - I've just finished a paper at SUTD about homomorphic computing. It is now available to read on arxiv here or from the journal here. Heres an article explaining ha its about and why this is going to be such an important topic in the coming years.

What is homomorphic computing?

Lets imagine a scenario: You are some person with some private data. Super private. You don't want anoyone knowing what it is. Now imagine you want to run an algorithm on your data to give you some ouptut. Maybe this will tell you whether your financial data is telling you its time for a merger, for example. But you don't have the ability to run the algorithm. Maybe its very intensive so you'd have to run it on the cloud using someone elses computer. Or maybe the algorithm has been developed by some other person who doesn't trust to give it to anyone else for fear they may copy it. What do you do?

Well - one solution is homorophic computing; here you would encrypt your data in a very special way. You have to encrypt it just that it is still malleable, you can still apply algorithms to the data and you will get some sort of encrypted answer which you can the decrypt. The encryption on the answer should only dependent on the encryption of the original data, and not the algorithm applied to it. That way people don't need to tell you what algorithm they applied to the data.
So the scheme woul work as follows.

  1. Encrypt your data
  2. Send this encrypted data to someone else to run
  3. Get some encrypted answer back
  4. Decrypt this to to get the answer
  5. Encrypt this to to get the answer
Easy peasy... right?

Is this even possible?

Well there has been a scheme from Gentry showing that it is possible to do with classical computers - however, as with alomst all classical encryption problems, its makes use of the assumption that certain problems are hard to slove (in this specific application it is related a shortest path problem on ideal lattices). But here in the quantum lab we really don't like making these kind of assumptions - instead aiming for information theoretic security. This is monstly due to the fact that if we get a quantum computer then many of these assumptions on the hardness of certain problems become false (for example RSA key distribution, a relative to Dickie-Helman encryption which forms the basis of many of our day to day protocols, gets broken by Shors Algorithm which finds the prime factors of a number efficiently).

However , there exists a no-go theorem that proves its impossible to do this efficiently and with perfect information protection even if you use a quantum computer. But this is only for perfect security- what if you leak just a tiny tiny bit of information? Or what is the compuation has some fintie probability of failure? Then that would still be pretty useful! Thats the idea with this paper - we describe an encryption scheme that could potentially leak a tiny amount of information about your data, but it allows computation to be preformed on the encrypted data.

So how do you do it?

This result is achieved by allowing the operation that correspond to computation commute with those that do encryption and decryption (this should be quite obvious when you think about how the scheme has to function). This is quite easy to accomplish in quantum mechanics by simply using the internal rotations on quantum states to be the encryption (like changing the basis of measurement - this is comparable to papers on random walks with encrypted data, and thus the boson sampling model of computation). The computation is then preformed on the spatial mode of the quantum objects - which could be, for example, acheived passively using linear optics.

And you can run any algorithm?

Well not exactly. To do this we would need to make sure that the operations the commute with the encryption/decryption from a universal set. The easy way to do this is to find a few operations which together are generators of a unversal set and make sure those commute. Now - this example allows a REALLY BIG set of operations. But to be honest I'm not clear about whether it's a universal set of operations. I orignally thought it should be, but it now appears that it isn't for group thoery reasons slightly beyond my grasp.

Does this paper have implications for further reseach?

I certianly believe so! This is the first paper (to my knowledge) on a quantum method for homomorphic computing - and I am aware of a few follow on methods for similiar papers which should be coming out of the reseach group soon. This method is partictularly important to emerging industries such as cloud computing - and improves upon other methods such a blind quantum computing by allowing operations to be preformed in a single round of communication between the client with the data and the server prefroming the hidden calculation.
